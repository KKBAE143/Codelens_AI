# CodeLens AI — Product Requirements Document (PRD)

### "Turn Any Codebase Into an Interactive Learning Experience"

**Version:** 1.0 | **Date:** March 30, 2026 | **Hackathon MVP**

---

## 1. Executive Summary

**CodeLens AI** transforms any GitHub repository, codebase, or technical knowledge base into beautiful, interactive courses — automatically. Think of it as "Notion meets Coursera, powered by AI, for codebases."

The original [codebase-to-course](https://github.com/zarazhangrui/codebase-to-course) is a Claude Code skill that generates a single HTML file from a repo. **CodeLens AI** elevates this into a full SaaS platform with multi-user collaboration, a course marketplace, enterprise onboarding workflows, and a persistent learning management system.

**One-Line Pitch:** *"Paste a GitHub URL → Get a stunning interactive course in minutes. No coding knowledge required."*

## 2. The Problem (Validated by Research)

### 2.1 Developer Onboarding Crisis

- **78% of new developers** find navigating an unfamiliar codebase challenging (JetBrains 2025 Survey).
- The average new engineer spends **2–3 weeks** just figuring out where things are — not building.
- **63% of remote workers** report feeling undertrained during onboarding.
- Companies expect a **3-month ramp-up** before new developers become fully productive.

### 2.2 Knowledge Loss from Employee Turnover

- When employees leave, companies lose institutional knowledge that can cost up to **213% of that individual's salary** to recover.
- **42% of the expertise** an employee uses is only known to them — undocumented, untransferable.
- **90% of organizations** believe retirements lead to significant knowledge and experience loss.
- After a senior employee leaves, team efficiency drops by **48%** and takes ~6 months to recover.

### 2.3 The Vibe Coding Explosion

- **92% of US-based developers** have adopted some form of AI-assisted coding (2026).
- **25% of Y Combinator W2025 startups** had codebases that were 95% AI-generated.
- The vibe coding tools market reached **$4.7B in 2025**, projected to grow to **$12.3B by 2027**.
- Critical problem: people can BUILD with AI but **cannot understand, debug, or maintain** what they built.
- A vibe-coded app suffered a **1.5M API key breach** because the developer hadn't written or understood a single line of code.

### 2.4 Existing Tools Don't Solve This

| Tool | What It Does | What It Misses |
|------|-------------|----------------|
| **Swimm** | Syncs docs to codebase via CI | No interactive learning, no courses, just documentation |
| **Mintlify** | Auto-generates API docs | Static documentation, not educational content |
| **DocuWriter.ai** | Generates code comments & UML | Developer-only, no non-technical audience support |
| **Sourcegraph Cody** | AI code assistant in IDE | Answers questions, doesn't teach systematically |
| **Traditional LMS (Udemy, Coursera)** | Hosts courses | Courses are manually created, never codebase-specific |

**The gap:** No product automatically generates interactive, visual, personalized learning courses from actual codebases. CodeLens AI fills this gap.

## 4. Product Vision & Features

### 4.1 Core Features (MVP — Hackathon Scope)

**F1: One-Click Course Generation**
- Paste a GitHub URL or connect a repo
- AI analyzes the entire codebase (structure, data flows, patterns)
- Generates a multi-module interactive course with:
  - Scroll-based navigation with progress tracking
  - Code ↔ Plain English side-by-side translations
  - Animated data flow visualizations
  - Interactive quizzes (scenario-based, not memorization)
  - Glossary tooltips on every technical term
  - Architecture diagrams

**F2: Course Dashboard**
- View all generated courses
- Track learning progress (modules completed, quiz scores)
- Share courses via link (public/private)
- Download as self-contained HTML

**F3: AI Chat Assistant (Contextual)**
- While viewing a course, ask follow-up questions
- "What happens if I add a new route here?"
- "How would I modify this to support multiple databases?"
- AI answers with full codebase context

**F4: Customization Engine**
- Choose target audience level (non-technical / junior dev / senior dev)
- Select focus areas (architecture, debugging, security, API usage)
- Custom branding (logo, colors) for enterprise/agency use

### 4.2 Growth Features (Post-MVP)

**F5: Course Marketplace**
- Users can publish generated courses publicly
- Browse courses by tech stack, framework, complexity
- "Top courses for React", "Understanding Django internals"

**F6: Team Workspaces**
- Organization-level accounts with team management
- Assign courses to new hires with deadlines
- Track team completion analytics
- Integration with Slack/Teams for course notifications

**F7: Knowledge Graph**
- Visual map of how concepts across courses connect
- "You learned about REST APIs in Course A — here's how GraphQL differs in Course B"

**F8: Auto-Update Courses**
- Webhook integration: when the repo changes, course auto-updates
- Diff view: "3 new files added since your last visit — here's what changed"

**F9: Multi-Format Export**
- Export as SCORM package (for enterprise LMS integration)
- Export as PDF workbook
- Export as slide deck (for workshops)

## 6. AI Pipeline — How Course Generation Works

### Phase 1: Repository Ingestion (30 seconds)
1. Clone repo via GitHub API (or accept ZIP upload)
2. Parse file tree, identify languages, frameworks
3. Read README, config files, entry points
4. Build dependency graph using Tree-sitter AST parsing

### Phase 2: Deep Analysis (1-2 minutes)
1. Identify the "cast of characters" (main components/modules)
2. Trace primary user journey end-to-end
3. Map data flows, API calls, communication patterns
4. Identify clever patterns (caching, error handling, auth)
5. Detect tech stack and framework conventions
6. Extract real code snippets (5-10 lines each) that teach key concepts

### Phase 3: Curriculum Design (30 seconds)
1. Structure 5-8 modules following the learning arc:
   - Module 1: "What this app does" + first user action traced
   - Module 2: Meet the components
   - Module 3: How pieces communicate
   - Module 4: External integrations (APIs, DBs)
   - Module 5: Clever engineering patterns
   - Module 6: When things break (debugging guide)
   - Module 7: Big picture architecture
2. For each module: design quizzes, select code snippets, create metaphors

### Phase 4: Content Generation (2-4 minutes)
1. Generate each module's content with AI:
   - Code ↔ Plain English translations
   - Unique metaphors per concept (NEVER repeat "restaurant")
   - Scenario-based quiz questions
   - Mermaid.js diagram specifications
   - Glossary terms with plain-English definitions
2. Assemble into responsive HTML with embedded CSS/JS
3. Add scroll-based navigation, progress tracking, keyboard nav
4. Generate animated visualizations (data flow, component chat)

### Phase 5: Quality Check (15 seconds)
1. Validate all code snippets match actual repo files
2. Ensure every module has at minimum: 1 quiz, 1 code translation, 1 visual
3. Check tooltip coverage (every technical term defined)
4. Verify responsive design and accessibility

**Total Generation Time: ~5-8 minutes per course**

## 7. Database Schema (Supabase/PostgreSQL)

```sql
-- Users (managed by Supabase Auth, extended here)
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  full_name TEXT,
  avatar_url TEXT,
  role TEXT DEFAULT 'free', -- free, pro, enterprise
  organization_id UUID REFERENCES organizations(id),
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Organizations (for team features)
CREATE TABLE organizations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  plan TEXT DEFAULT 'free',
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Courses
CREATE TABLE courses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  creator_id UUID REFERENCES profiles(id),
  organization_id UUID REFERENCES organizations(id),
  title TEXT NOT NULL,
  description TEXT,
  repo_url TEXT,
  repo_name TEXT,
  tech_stack TEXT[], -- ['react', 'node', 'postgres']
  target_audience TEXT DEFAULT 'beginner', -- beginner, intermediate, advanced
  module_count INT,
  estimated_time_minutes INT,
  html_file_path TEXT, -- Supabase Storage path
  thumbnail_url TEXT,
  is_public BOOLEAN DEFAULT false,
  status TEXT DEFAULT 'generating', -- generating, ready, failed
  generation_log JSONB,
  metadata JSONB, -- framework, language stats, etc.
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- Course Progress (per user)
CREATE TABLE course_progress (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id),
  course_id UUID REFERENCES courses(id),
  current_module INT DEFAULT 1,
  completed_modules INT[] DEFAULT '{}',
  quiz_scores JSONB DEFAULT '{}',
  last_accessed_at TIMESTAMPTZ DEFAULT now(),
  completed_at TIMESTAMPTZ,
  UNIQUE(user_id, course_id)
);

-- Chat Messages (AI assistant per course)
CREATE TABLE chat_messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  course_id UUID REFERENCES courses(id),
  user_id UUID REFERENCES profiles(id),
  role TEXT NOT NULL, -- 'user' or 'assistant'
  content TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Course Marketplace
CREATE TABLE course_likes (
  user_id UUID REFERENCES profiles(id),
  course_id UUID REFERENCES courses(id),
  PRIMARY KEY(user_id, course_id)
);

-- Waitlist / Early Access
CREATE TABLE waitlist (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL,
  source TEXT, -- 'landing', 'hackathon', 'referral'
  created_at TIMESTAMPTZ DEFAULT now()
);
```

4. **Interactive assessment** — scenario-based quizzes that test application, not memorization

## 12. Risk Mitigation

| Risk | Mitigation |
|------|-----------|
| AI generates low-quality courses | Extensive prompt engineering + quality validation pipeline + pre-tested on 10+ repos |
| Generation takes too long | Use streaming output + show progress + cache common patterns |
| Free tier AI rate limits | Queue system + fallback between Gemini/Groq/local models |
| Large repos exceed context window | Intelligent file selection — prioritize entry points, configs, key modules |
| Course looks generic | Custom design system (warm palette, bold typography) — NOT default AI aesthetics |
| GitHub API rate limits | Cache cloned repos, batch API calls, use authenticated requests |
